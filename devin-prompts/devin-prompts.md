## Task 1: 技術リスク・規約調査

### 目的
AndroidオーバーレイアプリをGoogle Playストアに出す前に、技術的・規約上のリスクを洗い出す。

### 調査項目

#### 1. SYSTEM_ALERT_WINDOW（オーバーレイ権限）の最新仕様
- Android 14 (API 34) でのユーザー付与制限（自動取り消しの条件）
- Android 15 (API 35) での挙動差
- Android 16 (API 36) での変更点（リリースノートを確認）
- 権限がない場合の代替動作（フォールバック）
- ストアの権限審査基準

#### 2. Foreground Service 制約
- Android 14+: `foregroundServiceType` の指定ルール
- Android 15+: フォアグラウンドサービスのタイムアウト制限（onTimeout）
- `FOREGROUND_SERVICE_SPECIAL_USE` の宣言方法
- バッテリー最適化の影響（Doze/App Standby Bucket）
- ユーザーによる手動停止時の挙動

#### 3. Google Play ポリシー
- オーバーレイ権限を持つアプリの審査基準（2024-2025年版）
- 不適切なオーバーレイ使用事例（広告バナー、偽ボタンなど）
- プライバシーポリシー必須条件
- ユーザーデータに関する宣言
- 年齢区分の選択

#### 4. 最新ストア状況
- 2025年〜2026年のオーバーレイアプリ関連のポリシー改定
- 直近でリジェクトされたオーバーレイアプリの事例

## Task 2: 競合調査

### 目的
類似アプリの成功要因と失敗要因を分析し、差別化ポイントを確定する。

### 調査対象カテゴリ

1. **Android デスクトップペット系**
   - 例: いぬっこ（Android Pet）、Desktop Pet、Pocket Petなど
   - 検索キーワード: "Android overlay pet", "Android desktop pet", "android 画面端 ペット"
   - 調査項目: DL数、評価、最近のアップデート、レビュー内容、権限に対する不安

2. **オーバーレイ常駐キャラクター系**
   - 例: おしゃべりキャラ、画面上で動くマスコット
   - 低評価理由: 広告多すぎ、バッテリー消費、邪魔、説教くさい

3. **バーチャルペット・育成系**
   - AndroidのTalking Tom類似、Pou系、放置育成系
   - UIパターン、育成システムの設計

4. **AIキャラチャット系**
   - Character.AIアプリ、スマホ常駐型のAI相棒アプリ
   - ただしAIなしの部分のみ参考（API通信しないという独自性を評価）

5. **類似コンセプト（海外）**
   - FizzPop、Bubble Friendなど日本語で見つけにくいものも含む

### 各アプリの調査項目
- パッケージ名 / ストアURL
- 推定DL数
- 星評価 (★x.x / 5)
- 直近のレビュー傾向（低評価理由のトップ3）
- 権限への言及（怖い・邪魔・プライバシー不安）
- 収益化モデル
- スクリーンショット/動画の表現
- 最終アップデート日

## Task 4: QA チェックリスト設計

### 目的
実機検証用の総合QAチェックリストを作成する。

### 検証カテゴリ

#### 権限・初回起動
- [ ] 初回起動時に権限説明画面が表示される
- [ ] 権限拒否時のハンドリングが適切（説明→再度促す）
- [ ] 権限許可後、OverlayServiceが起動する
- [ ] 権限許可後、勇者Viewが表示される
- [ ] 設定画面に遷移できる

#### オーバーレイ基本
- [ ] 他アプリ（Chrome, X, Instagram, YouTube）上で表示される
- [ ] ホーム画面で表示される
- [ ] ロック画面で非表示（または意図した挙動）
- [ ] ドラッグで移動できる
- [ ] 画面端に吸着する
- [ ] 最小化ボタンで小さくなる
- [ ] 最小化状態をタップで元に戻る
- [ ] 閉じるボタンで完全に非表示
- [ ] 設定画面から再表示できる

#### 台詞・リアクション
- [ ] タップで吹き出しが表示される
- [ ] 3.5秒後に吹き出しが消える
- [ ] 連続タップでも短時間に複数表示されない
- [ ] 時間帯で台詞が変わる
- [ ] バッテリー低下で台詞が変わる
- [ ] 放置復帰で台詞が変わる
- [ ] 設定「少ない/普通/多い」で頻度が変わる

#### 永続化
- [ ] アプリ再起動後も設定が保持される
- [ ] アプリ再起動後もレベル/経験値が保持される
- [ ] データ全消去でリセットされる

#### Androidバージョン別
- [ ] Android 8.0 (API 26) で動作
- [ ] Android 10 (API 29) で動作
- [ ] Android 12 (API 31) で動作
- [ ] Android 13 (API 33) で動作（通知権限）
- [ ] Android 14 (API 34) で動作
- [ ] Android 15 (API 35) で動作

#### ストレステスト
- [ ] 24時間常駐させてメモリリークなし
- [ ] 1時間あたりのバッテリー消費1%以下
- [ ] 画面回転でクラッシュしない
- [ ] マルチウィンドウモードでクラッシュしない
- [ ] 他のオーバーレイアプリと共存できる

#### 敵認定テスト（避けるべき動作）
- [ ] 決済画面で邪魔にならない
- [ ] 全画面動画再生中でも動作が軽い
- [ ] キーボード表示中に不自然な挙動をしない
- [ ] 通知内容を読み取っていないことが外見からもわかる

- QAチェックリストMarkdownファイル
- 実機検証ログテンプレート（インストール日、端末、Androidバージョン、OK/NG項目）

## Task 5: ストア提出前チェックリスト

### 目的
Google Playストア提出前に必要な資料とチェック項目をまとめる。

### 提出前に作成するもの
- [ ] プライバシーポリシー（Markdown / Webページ）
- [ ] ストア説明文（タイトル・短い説明・長い説明）
- [ ] スクリーンショット方針（何を写すか）
- [ ] 権限説明文（ユーザー向け）

### ストア説明文ドラフト
以下を埋める形でドラフトを作成。

```
## タイトル（30字以内）
画面端クエスト - スマホに住む小さな勇者

## 短い説明（80字以内）
スマホ画面の端に住む、小さな勇者。タップすると喋る。時間帯やバッテリーで反応が変わる。

## 長い説明
（300〜400字で作成）

## スクリーンショット
何を写すべきか：
1. 画面端に勇者がいる様子
2. タップで吹き出し
3. 設定画面
4. 育成進化
5. 時間帯によって変わる反応
```

### プライバシーポリシー草案
以下を含む簡易プライバシーポリシー草案を作成。
- 取得する情報（バッテリー残量、時間帯のみ）
- 取得しない情報（通知本文、画面内容、位置情報、連絡先、SMS、写真、カメラ、マイク）
- 第三者提供の有無
- データ保存場所（端末内のみ、クラウド保存なし）
- 連絡先
- 改定日

### 提出前チェック項目
- [ ] プライバシーポリシーのURLが有効
- [ ] 権限の説明がストアとアプリ内で一致している
- [ ] オーバーレイ権限の利用理由が明確
- [ ] 子どものプライバシーに関する設定
- [ ] 広告の有無とそのSDK
- [ ] データセーフティセクションの内容が実装と一致
- [ ] 対応端末一覧に制限がないか

### 成果物
- ストア提出チェックリスト
- プライバシーポリシー草案（英語＋日本語）
- ストア説明文ドラフト
